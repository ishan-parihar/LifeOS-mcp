# LifeOS Unified Database Schema Upgrade Report

**Generated:** 2026-04-08
**Architect:** Master Schema Synthesizer
**Inputs:** 7 C-Suite Agent Proposals (CEO, COO, CPO, CRO, CFO, CMO, Physician)
**Scope:** 23 existing databases → 28 databases (+5 new), ~450+ total properties

---

## 1. Executive Summary

### What Changes
LifeOS currently has **23 databases** with ~230 total properties, many of which are under-structured (JSON blobs, free-text fields, redundant flags). The 7 C-suite agents independently proposed **~500 property additions**, **~40 property removals**, and **5 new databases**. After merging, deconflicting, and eliminating duplicates, the unified upgrade delivers:

- **28 databases** (23 existing + 5 new)
- **~440 total properties** across all databases
- **~80 formula properties** (computed intelligence)
- **~55 rollup properties** (cross-database aggregation)
- **~50 relation properties** added (complete interconnectivity)
- **~35 properties removed** (noise elimination)

### Why
The current schema has critical gaps:
1. **Unstructured blobs** — `nutrition` (diet), `psychograph` (mood), `financial_json`, `diet_json`, `drl_json`, `systemic_json`, `week_json`, `month_json`, `day_json` — data trapped in text fields that agents cannot query, filter, or compute against.
2. **Missing relations** — `activity_type` is a Select instead of Relation, preventing rollups. No relation from activity_log → diet_log, or financial_log → campaigns.
3. **No health/budget/asset tracking** — Vital signs, budget lines, and financial accounts don't exist as databases.
4. **Redundant fields** — `monitor` appears in 3 databases with unclear purpose. `habit` duplicated between activity_log and activity_types. `id` auto-generated by Notion, stored explicitly.

### Impact
- **Query speed:** Structured properties replace LLM calls for every analysis (JSON blobs → typed fields)
- **Agent autonomy:** Every formula is pre-computed; agents read signals, not raw data
- **Cross-domain intelligence:** 50+ new relations enable correlation (e.g., mood ↔ project load, revenue ↔ activity type)
- **Scalability:** New databases (health_interventions, financial_accounts, budget_lines, network_map, social_events) are leaf-level foundations for future agent capabilities

## 2. Per-Database Unified Schema

### Ownership Convention
Each property lists the **owning agent** who reads/writes it primarily:
- **CEO** = Strategic oversight, decision-making
- **COO** = Operational execution, scheduling
- **CPO** = Psychological/emotional layer
- **CRO** = Relational/network layer
- **CFO** = Financial/capital layer
- **CMO** = Content/marketing layer
- **PHY** = Physician/health layer
- **SYS** = System-level (all agents read)
