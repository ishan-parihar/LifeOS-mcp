# LifeOS MCP — R&D Summary (v0.5.0) — 2026-04-01

## Scope
This report documents all changes made during the recent R&D cycle to make LifeOS MCP agent-ready with reliable startup context and operational content workflows — without tool bloat.

## Highlights
- New agent-scoped Context Cards (lifeos_context_card) for reliable session bootstrap
- First-class domain tools where workflow guardrails matter (Content, Campaigns)
- Corrected property mappings for content metrics and media assets
- Version bump and discoverability improvements

## Versioning
- MCP Server: v0.5.0 (src/index.ts)
- New tools registered: lifeos_context_card, lifeos_content, lifeos_campaigns

## Added — Agent Context Bootstrap
- Tool: lifeos_context_card (src/tools/context-card.ts)
  - Purpose: Compact, deterministic startup context per agent role
  - Detail levels: compact | schemas | workflows | all
  - Covers: role fit, agent-scoped DB map, tool routing, property conventions, top recipes
- Discover updated to list the bootstrap tool (src/tools/discover.ts)

## Added — Content Pipeline Operations
- Tool: lifeos_content (src/tools/content.ts)
  - Actions:
    - list: filter by status/date; quick overview
    - transition: guarded status changes (override optional); canonical pipeline respected
    - publish: mark Published 💥 and set live_url(s)
    - update_metrics: bulk write reach/clicks/engagement
    - calendar: render calendar view for a date range
- Transformers: src/transformers/content.ts
  - transformContent, contentsToMarkdown, calendarMarkdown
- Status transition matrix enforced (canonical order):
  Potential Idea → Scheduled → Next Up 🚩 → Writing 📝 → Recording ⏺ → Editing 🎞 → Ready to Post 📤 → Published 💥

## Added — Campaign Operations
- Tool: lifeos_campaigns (src/tools/campaigns.ts)
  - Actions:
    - list: window/frequency/platform filters
    - brief: single-campaign brief by page_id
- Transformers: src/transformers/campaigns.ts
  - transformCampaign, campaignsToMarkdown

## Fixed — Property Mappings
- Writable numbers for content metrics (src/tools/entry-helpers.ts):
  - reach, clicks, engagement now supported as number fields
- Media assets support (files via external URLs):
  - media_assets → "Media Assets" (files) formatted as external URLs
- Formula skip-list corrected so reach/clicks/engagement are not treated read-only

## Notion Client Enhancements
- getPage(pageId) added for single-record fetch (src/notion/client.ts)

## Discover / Docs
- Discover now lists: lifeos_context_card, lifeos_campaigns, lifeos_content (src/tools/discover.ts)

## Psychologist Session — Validation
- Queried last-week journals (subjective/relational/systemic) and saved a session report via lifeos_create_report
- Confirms journaling + reporting loop operational alongside new bootstrap

## Usage Examples
- Bootstrap (agent-aware):
  - lifeos_context_card(agent='content', detail='compact')
  - lifeos_context_card(agent='strategic', detail='compact')
- Content:
  - lifeos_content(action='list', status='Next Up 🚩', limit=20)
  - lifeos_content(action='transition', content_id='<id>', to='Writing 📝')
  - lifeos_content(action='publish', content_ids=['<id>'], live_urls=['https://...'])
  - lifeos_content(action='update_metrics', content_ids=['<id>'], reach=1200, clicks=45, engagement=210)
  - lifeos_content(action='calendar', date_from='YYYY-MM-DD', date_to='YYYY-MM-DD')
- Campaigns:
  - lifeos_campaigns(action='list', start_date_from='YYYY-MM-DD', end_date_to='YYYY-MM-DD')
  - lifeos_campaigns(action='brief', campaign_id='<id>')

## Design Decisions
- Avoid per-database tools; use generic CRUD + context cards for most DBs
- Domain tools only where guardrails and sequencing add value (content/campaigns)
